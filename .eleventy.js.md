# Eleventy Configuration Guide

This document explains the configuration choices made in `.eleventy.js` and how to extend them.

## Collections

### Blog Collection
- **Path**: `content/blog/**/*.md`
- **Filter**: Only published posts (`published !== false`)
- **Sort**: Reverse chronological (newest first)

### Featured Posts Collection
- **Path**: `content/blog/**/*.md`
- **Filter**: Published AND featured posts
- **Sort**: Reverse chronological
- **Usage**: Homepage featured section

### Projects Collection
- **Path**: `content/projects/**/*.md`
- **Sort**: By `order` field (ascending)

### Publications Collection
- **Path**: `content/publications/**/*.md`
- **Sort**: By `year` field (descending)

### Media Collection
- **Path**: `content/media/**/*.md`
- **Sort**: Reverse chronological by date

### Newsletter Collection
- **Path**: `content/newsletter/**/*.md`
- **Sort**: Reverse chronological by date

### Tag List Collection
- Automatically generates list of all unique tags used across content
- Used for tag archive pages

## Custom Filters

### Date Filters
- `readableDate`: "January 15, 2025"
- `htmlDateString`: "2025-01-15" (for `<time>` elements)
- `isoDate`: ISO 8601 format for RSS feeds
- `yearFromDate`: "2025" (for copyright, etc.)

### Content Filters
- `readingTime`: Calculates reading time at 200 words/minute
- `excerpt`: Generates excerpt from content if not in frontmatter
- `limit`: Limits array to specified number of items
- `relatedPosts`: Finds related posts based on shared tags

### Collection Filters
- `filterByTag`: Filters posts by specific tag

## Markdown Configuration

Uses `markdown-it` with:
- HTML enabled
- Auto-linking of URLs
- Typographer (smart quotes, em dashes)
- Anchor links on headings (H2-H4)

## Passthrough Copy

These files/folders are copied directly to output without processing:
- `src/assets/` → Static assets (CSS, images, favicon)
- `CNAME` → GitHub Pages custom domain
- `content/assets/` → Content-specific assets
- `src/robots.txt` → SEO/crawler configuration

## Directory Structure

- **Input**: Root directory
- **Includes**: `src/_includes/`
- **Layouts**: `src/_layouts/`
- **Data**: `src/_data/`
- **Output**: `_site/`

## Template Engines

- Markdown files: Processed through Nunjucks
- HTML files: Processed through Nunjucks
- Data files: Processed through Nunjucks

This allows using Nunjucks syntax (variables, loops, conditionals) in all content files.
